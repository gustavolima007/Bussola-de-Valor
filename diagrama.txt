Passo a passo para gerar o diagrama do projeto no Napkin AI:

1. Acesse o site https://napkin.ai/ e faça login ou crie uma conta gratuita.

2. Clique em "New Diagram" ou similar para criar um novo diagrama.

3. No campo de descrição de texto, cole o seguinte prompt detalhado sobre a arquitetura do projeto "Bússola de Valor":

---

Descrição da Arquitetura do Sistema "Bússola de Valor":

Este é um dashboard interativo para análise fundamentalista de ações da B3, focado em investimentos em dividendos. A arquitetura segue um pipeline de dados ETL, armazenamento em data warehouse e apresentação via dashboard web.

Componentes Principais:

- Fontes de Dados Externas:
  - yfinance API: Para dados históricos de preços, dividendos e indicadores.
  - brapi API: Para dados em tempo real da B3, incluindo ações e fundos.

- Módulo Data Engineer (ETL Pipeline):
  - Scripts Python numerados (01-acoes_e_fundos.py até 12-indices.py) que executam extração, transformação e carga de dados.
  - Orquestrador (loader.py) que executa os scripts em sequência.

- Data Warehouse (DuckDB):
  - Land DW: Camada de aterrissagem com dados brutos em Parquet.
  - Trusted DW: Camada confiável com dados limpos e transformados.
  - Banco DW: Camada analítica com agregações e métricas finais.

- Aplicação Streamlit:
  - app.py: Ponto de entrada principal.
  - components/: Módulos para filtros, abas e calculadora.
  - data_loader.py: Carrega dados do DuckDB para o dashboard.
  - scoring.py: Lógica de pontuação das ações baseada em critérios fundamentalistas.
  - styles/: CSS para estilização.

- Infraestrutura Suporte:
  - GitHub Repository: Versionamento de código.
  - GitHub Actions: Workflow automatizado para atualização diária do DW às 07:00.
  - Supabase: Opção futura para banco relacional (atualmente usado em HML para testes).
  - HML/: Ambiente de homologação para testes de novas funcionalidades.

Fluxos de Dados:

- Extração: Fontes Externas -> Scripts ETL (Data Engineer) -> Land DW (DuckDB).
- Transformação: Land DW -> Scripts ETL -> Trusted DW (DuckDB).
- Carga Analítica: Trusted DW -> Scripts ETL -> Banco DW (DuckDB).
- Apresentação: Banco DW -> data_loader.py -> Streamlit App -> Usuário via navegador.

Pontuações e Lógica de Negócios:

- Score de Ações: Calculado baseado em Dividend Yield, Valuation (P/L, P/VP), Rentabilidade (ROE), Saúde Financeira, etc., totalizando até 1000 pontos.
- Score de Setores: Média dos scores das ações, com bônus/penalidades por subsetores.
- Dashboard: Exibe rankings, gráficos interativos com Plotly, filtros por setor, porte, etc.

Gere um diagrama de arquitetura mostrando os componentes conectados com setas indicando o fluxo de dados, destacando as camadas do DW e a separação entre ETL, armazenamento e apresentação.

---

4. Após colar o prompt, clique em "Generate Diagram" ou botão equivalente no Napkin AI.

5. O Napkin AI irá gerar automaticamente um diagrama visual baseado na descrição. Você pode ajustar, adicionar detalhes ou exportar a imagem.

6. Salve ou compartilhe o diagrama gerado para documentar a arquitetura do projeto.
